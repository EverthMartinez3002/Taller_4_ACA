    API de Pedidos#

    La API de pedidos gestiona todo el ciclo de vida de los pedidos, desde la creación hasta la entrega, incluyendo pagos, seguimiento y gestión administrativa.

    Base URL#

    https://api.cupcakeapi.com/v1/orders

    🛒 Creación de Pedidos#

    📋 Gestión de Pedidos#

    Listar Pedidos del Usuario#

    Obtiene el historial de pedidos del usuario autenticado.

    GET /orders
Authorization: Bearer JWT_TOKEN

    Parámetros de Query: - status (string): Filtrar por estado - start_date (date): Fecha inicio (YYYY-MM-DD) - end_date (date): Fecha fin (YYYY-MM-DD) - page (int): Número de página - limit (int): Elementos por página (max: 50) - sort (string): Ordenar por (created_at, total, status) - order (string): Dirección (asc, desc)

    Estados Posibles: - pending - Pendiente de confirmación - confirmed - Confirmado y pagado - preparing - En preparación - ready - Listo para entrega/recogida - out_for_delivery - En camino - delivered - Entregado - picked_up - Recogido - cancelled - Cancelado - refunded - Reembolsado

    Respuesta (200): 

    {
    "status": "success",
    "data": {
        "orders": [
            {
                "id": "ORD-2024-001234",
                "order_number": "CK240120001",
                "status": "delivered",
                "created_at": "2024-01-20T10:15:00Z",
                "delivered_at": "2024-01-25T15:22:00Z",
                "total": 86.71,
                "items_count": 18,
                "delivery_type": "delivery",
                "can_reorder": true,
                "can_review": true
            }
        ]
    },
    "pagination": {
        "current_page": 1,
        "total_pages": 3,
        "total_items": 28
    }
}

    Obtener Pedido Específico#

    Obtiene todos los detalles de un pedido específico.

    GET /orders/{order_id}
Authorization: Bearer JWT_TOKEN

    Reordenar#

    Crea un nuevo pedido basado en uno anterior.

    POST /orders/{order_id}/reorder
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición (Opcional): 

    {
    "exclude_items": [2], // IDs de items a excluir
    "delivery": {
        "type": "pickup",
        "scheduled_datetime": "2024-02-01T14:00:00Z"
    },
    "modifications": {
        "items": [
            {
                "item_id": 1,
                "quantity": 12, // Cambiar cantidad
                "customizations": {
                    "frosting": "chocolate_ganache"
                }
            }
        ]
    }
}

    ✏️ Modificación de Pedidos#

    Modificar Pedido#

    Modifica un pedido existente (solo si está permitido).

    PUT /orders/{order_id}
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "items": [
        {
            "action": "update",
            "item_id": 1,
            "quantity": 8,
            "customizations": {
                "decoration": "edible_flowers"
            }
        },
        {
            "action": "add",
            "cupcake_id": 5,
            "quantity": 6,
            "customizations": {
                "frosting": "lemon_buttercream"
            }
        },
        {
            "action": "remove",
            "item_id": 2
        }
    ],
    "delivery": {
        "scheduled_datetime": "2024-01-25T16:00:00Z",
        "delivery_instructions": "Llamar al llegar, no tocar timbre"
    },
    "modification_reason": "Cliente solicitó cambios en la celebración"
}

    Cancelar Pedido#

    Cancela un pedido existente.

    DELETE /orders/{order_id}
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "reason": "changed_plans", // "changed_plans", "wrong_order", "delivery_issue", "other"
    "details": "La celebración se pospuso para la próxima semana",
    "request_refund": true
}

    Respuesta (200): 

    {
    "status": "success",
    "data": {
        "order": {
            "id": "ORD-2024-001234",
            "status": "cancelled",
            "cancelled_at": "2024-01-22T14:30:00Z",
            "cancellation_reason": "changed_plans"
        },
        "refund": {
            "amount": 86.71,
            "method": "original_payment_method",
            "estimated_processing_time": "3-5 días hábiles",
            "refund_id": "ref_abc123xyz"
        }
    }
}

    🔄 Gestión Administrativa (Staff)#

    Listar Todos los Pedidos#

    Lista pedidos para administración. Requiere permisos de employee o superior.

    GET /admin/orders
Authorization: Bearer JWT_TOKEN

    Parámetros de Query: - status (string): Filtrar por estado - customer_id (int): Filtrar por cliente - date_range (string): Rango de fechas - delivery_type (string): Tipo de entrega - payment_status (string): Estado del pago - priority (string): Prioridad (normal, urgent, rush) - assigned_to (int): Asignado a empleado - search (string): Buscar por número de orden o cliente

    Actualizar Estado de Pedido#

    Cambia el estado de un pedido.

    PATCH /admin/orders/{order_id}/status
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "status": "preparing",
    "estimated_completion": "2024-01-25T14:00:00Z",
    "assigned_to": 456, // ID del empleado
    "internal_notes": "Cliente solicitó extra decoración",
    "notify_customer": true
}

    Asignar Pedido#

    Asigna un pedido a un empleado específico.

    POST /admin/orders/{order_id}/assign
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "employee_id": 456,
    "role": "baker", // "baker", "decorator", "driver"
    "priority": "normal",
    "estimated_completion": "2024-01-25T14:00:00Z",
    "notes": "Cliente VIP, prestar atención especial a la decoración"
}

    Programar Entrega#

    Programa o reprograma la entrega de un pedido.

    PUT /admin/orders/{order_id}/delivery/schedule
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "driver_id": 789,
    "scheduled_datetime": "2024-01-25T15:30:00Z",
    "route_order": 3,
    "estimated_duration": 20,
    "special_instructions": "Edificio tiene estacionamiento limitado"
}

    💰 Gestión de Pagos#

    Procesar Reembolso#

    Procesa un reembolso para un pedido.

    POST /admin/orders/{order_id}/refund
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "amount": 86.71, // Monto a reembolsar
    "reason": "quality_issue",
    "description": "Cliente reportó que los cupcakes llegaron dañados",
    "refund_method": "original_payment", // "original_payment", "store_credit", "cash"
    "notify_customer": true,
    "internal_notes": "Revisar proceso de empaque"
}

    Aplicar Descuento Posterior#

    Aplica un descuento o crédito después de la compra.

    POST /admin/orders/{order_id}/discount
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "type": "percentage", // "percentage", "fixed_amount", "store_credit"
    "value": 20,
    "reason": "service_recovery",
    "description": "Compensación por retraso en la entrega",
    "apply_as": "store_credit" // "refund", "store_credit", "future_discount"
}

    🚨 Gestión de Problemas#

    Reportar Problema#

    Permite reportar un problema con un pedido.

    POST /orders/{order_id}/issues
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "type": "quality", // "quality", "delivery", "missing_items", "wrong_order", "other"
    "severity": "medium", // "low", "medium", "high", "critical"
    "title": "Cupcakes llegaron dañados",
    "description": "Algunos cupcakes se volcaron durante el transporte y la decoración se arruinó",
    "affected_items": [1, 2],
    "images": [
        "data:image/jpeg;base64,/9j/4AAQSkZJRgABAQAAAQ..."
    ],
    "requested_resolution": "replacement", // "refund", "replacement", "credit", "none"
    "contact_preference": "phone"
}

    Resolver Problema#

    Marca un problema como resuelto.

    PUT /admin/orders/{order_id}/issues/{issue_id}/resolve
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "resolution": "replacement_sent",
    "description": "Se envió reemplazo completo del pedido sin costo adicional",
    "compensation": {
        "type": "store_credit",
        "amount": 25.00
    },
    "follow_up_required": false,
    "internal_notes": "Revisar protocolo de empaque con equipo de preparación"
}

    Códigos de Error Específicos#

    Código Descripción Solución
    ORDER_NOT_FOUND Pedido no encontrado Verificar ID del pedido
    ORDER_CANNOT_BE_MODIFIED Pedido no puede modificarse Verificar estado y tiempo límite
    ORDER_ALREADY_CANCELLED Pedido ya fue cancelado No se puede realizar la acción
    PAYMENT_FAILED Error en el pago Verificar método de pago
    DELIVERY_ADDRESS_INVALID Dirección de entrega inválida Verificar formato de dirección
    ITEMS_OUT_OF_STOCK Productos sin stock Seleccionar productos disponibles
    DELIVERY_TIME_UNAVAILABLE Horario no disponible Seleccionar otra fecha/hora
    ORDER_TOTAL_BELOW_MINIMUM Total por debajo del mínimo Agregar más productos
    COUPON_INVALID Cupón inválido o expirado Verificar código y validez
    LOYALTY_POINTS_INSUFFICIENT Puntos insuficientes Verificar balance de puntos

    La API de pedidos proporciona un sistema completo para gestionar todo el ciclo de vida de los pedidos, desde la creación hasta la entrega, con funcionalidades tanto para clientes como para administradores.
