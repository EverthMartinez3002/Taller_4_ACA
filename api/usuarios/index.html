    API de Usuarios#

    La API de usuarios permite gestionar cuentas de usuario, perfiles, autenticación y preferencias en CupcakeAPI.

    Base URL#

    https://api.cupcakeapi.com/v1/users

    📝 Registro de Usuario#

    Crear Nueva Cuenta#

    Registra un nuevo usuario en el sistema.

    POST /auth/register
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "email": "maria@ejemplo.com",
    "password": "contraseña123",
    "name": "María García",
    "phone": "+1234567890",
    "birth_date": "1990-05-15",
    "preferences": {
        "newsletter": true,
        "sms_notifications": false
    }
}

    Respuesta Exitosa (201): 

    {
    "status": "success",
    "data": {
        "user": {
            "id": 12345,
            "email": "maria@ejemplo.com",
            "name": "María García",
            "phone": "+1234567890",
            "birth_date": "1990-05-15",
            "role": "customer",
            "verified": false,
            "loyalty_points": 0,
            "created_at": "2024-01-20T10:30:00Z"
        },
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "refresh_token": "rt_1234567890abcdef",
        "expires_in": 3600
    }
}

    Errores Comunes: - 422 - Email ya registrado - 422 - Contraseña muy débil - 422 - Formato de email inválido

    🔐 Autenticación#

    Iniciar Sesión#

    Autentica un usuario existente.

    POST /auth/login
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "email": "maria@ejemplo.com",
    "password": "contraseña123",
    "remember_me": true
}

    Respuesta Exitosa (200): 

    {
    "status": "success",
    "data": {
        "user": {
            "id": 12345,
            "email": "maria@ejemplo.com",
            "name": "María García",
            "role": "customer",
            "verified": true,
            "last_login": "2024-01-20T10:30:00Z"
        },
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "refresh_token": "rt_1234567890abcdef",
        "expires_in": 3600
    }
}

    Renovar Token#

    Obtiene un nuevo token de acceso usando el refresh token.

    POST /auth/refresh
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "refresh_token": "rt_1234567890abcdef"
}

    Cerrar Sesión#

    Invalida el token actual.

    POST /auth/logout
Authorization: Bearer JWT_TOKEN

    Verificar Email#

    Verifica la dirección de email del usuario.

    POST /auth/verify-email
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "verification_code": "123456"
}

    Actualizar Perfil#

    Actualiza la información del usuario.

    PUT /users/me
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "name": "María García López",
    "phone": "+1234567891",
    "bio": "Pastelera aficionada y adicta a los cupcakes",
    "preferences": {
        "favorite_flavors": ["chocolate", "strawberry", "lemon"],
        "dietary_restrictions": ["vegetarian", "nut_free"],
        "newsletter": true,
        "sms_notifications": true
    }
}

    Cambiar Contraseña#

    Cambia la contraseña del usuario.

    PUT /users/me/password
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "current_password": "contraseña123",
    "new_password": "nuevaContraseña456",
    "confirm_password": "nuevaContraseña456"
}

    Subir Foto de Perfil#

    Sube una nueva imagen de perfil.

    POST /users/me/avatar
Authorization: Bearer JWT_TOKEN
Content-Type: multipart/form-data

    Cuerpo de la Petición: 

    Content-Disposition: form-data; name="avatar"; filename="perfil.jpg"
Content-Type: image/jpeg

[binary image data]

    Respuesta (200): 

    {
    "status": "success",
    "data": {
        "profile_image": "https://images.cupcakeapi.com/users/12345.jpg",
        "thumbnail": "https://images.cupcakeapi.com/users/12345_thumb.jpg"
    }
}

    🏠 Gestión de Direcciones#

    Listar Direcciones#

    Obtiene todas las direcciones del usuario.

    GET /users/me/addresses
Authorization: Bearer JWT_TOKEN

    Agregar Dirección#

    Agrega una nueva dirección al perfil.

    POST /users/me/addresses
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "type": "work",
    "label": "Oficina",
    "street": "Av. Empresarial 456",
    "apt_suite": "Piso 3, Oficina 301",
    "city": "Ciudad Trabajo",
    "state": "Estado",
    "zip_code": "54321",
    "country": "País",
    "delivery_instructions": "Recepción en planta baja",
    "is_default": false
}

    Actualizar Dirección#

    Modifica una dirección existente.

    PUT /users/me/addresses/{address_id}
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Eliminar Dirección#

    Elimina una dirección del perfil.

    DELETE /users/me/addresses/{address_id}
Authorization: Bearer JWT_TOKEN

    Establecer Dirección por Defecto#

    Marca una dirección como predeterminada.

    POST /users/me/addresses/{address_id}/set-default
Authorization: Bearer JWT_TOKEN

    🔒 Recuperación de Contraseña#

    Solicitar Recuperación#

    Envía un email con instrucciones para restablecer la contraseña.

    POST /auth/forgot-password
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "email": "maria@ejemplo.com"
}

    Restablecer Contraseña#

    Establece una nueva contraseña usando el código de recuperación.

    POST /auth/reset-password
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "email": "maria@ejemplo.com",
    "reset_code": "ABC123XYZ",
    "new_password": "nuevaContraseña789",
    "confirm_password": "nuevaContraseña789"
}

    🗑️ Eliminar Cuenta#

    Solicitar Eliminación#

    Inicia el proceso de eliminación de cuenta.

    POST /users/me/delete-request
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "reason": "Ya no uso el servicio",
    "password": "contraseñaActual"
}

    Cancelar Eliminación#

    Cancela una solicitud de eliminación pendiente.

    DELETE /users/me/delete-request
Authorization: Bearer JWT_TOKEN

    Confirmar Eliminación#

    Confirma definitivamente la eliminación de la cuenta.

    POST /users/me/delete-confirm
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "confirmation_code": "DELETE123",
    "password": "contraseñaActual"
}

    📱 Notificaciones#

    Obtener Configuración#

    Consulta las preferencias de notificaciones.

    GET /users/me/notifications/settings
Authorization: Bearer JWT_TOKEN

    Actualizar Configuración#

    Modifica las preferencias de notificaciones.

    PUT /users/me/notifications/settings
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "email": {
        "order_confirmations": true,
        "delivery_updates": true,
        "promotions": false,
        "newsletter": true
    },
    "sms": {
        "order_confirmations": true,
        "delivery_updates": true,
        "promotions": false
    },
    "push": {
        "order_confirmations": true,
        "delivery_updates": true,
        "promotions": true,
        "new_flavors": true
    }
}

    Listar Notificaciones#

    Obtiene las notificaciones recientes del usuario.

    GET /users/me/notifications
Authorization: Bearer JWT_TOKEN

    Parámetros de Query: - page (int): Número de página - limit (int): Elementos por página (max: 50) - unread_only (boolean): Solo notificaciones no leídas

    Marcar como Leída#

    Marca una notificación específica como leída.

    PUT /users/me/notifications/{notification_id}/read
Authorization: Bearer JWT_TOKEN

    Marcar Todas como Leídas#

    Marca todas las notificaciones como leídas.

    PUT /users/me/notifications/read-all
Authorization: Bearer JWT_TOKEN

    Códigos de Error Específicos#

    Código Descripción Solución
    USER_NOT_FOUND Usuario no encontrado Verificar ID del usuario
    EMAIL_ALREADY_EXISTS Email ya registrado Usar email diferente o recuperar cuenta
    WEAK_PASSWORD Contraseña muy débil Usar contraseña más fuerte
    INVALID_VERIFICATION_CODE Código de verificación inválido Solicitar nuevo código
    ADDRESS_LIMIT_EXCEEDED Máximo de direcciones alcanzado Eliminar direcciones no utilizadas
    PAYMENT_METHOD_INVALID Método de pago inválido Verificar información de la tarjeta
    INSUFFICIENT_LOYALTY_POINTS Puntos insuficientes Acumular más puntos antes de canjear

    La API de usuarios proporciona todas las funcionalidades necesarias para gestionar cuentas de usuario de manera completa y segura. Asegúrate de implementar validaciones apropiadas en el frontend y manejar todos los casos de error correctamente.
