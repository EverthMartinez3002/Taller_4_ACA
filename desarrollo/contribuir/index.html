    Guía para Contribuir#

    ¡Gracias por tu interés en contribuir al ecosistema de CupcakeAPI! Esta guía te ayudará a entender cómo puedes participar y mejorar la plataforma.

    🌟 Formas de Contribuir#

    🐛 Reportar Errores#

    • Reporta bugs y problemas de la API
    • Sugiere mejoras en la documentación
    • Identifica problemas de seguridad

    💡 Proponer Funcionalidades#

    • Nuevos endpoints de API
    • Mejoras en endpoints existentes
    • Integraciones con servicios externos

    📖 Mejorar Documentación#

    • Corregir errores tipográficos
    • Agregar ejemplos de código
    • Traducir documentación

    🔧 Contribuir con Código#

#### Branching Strategy
```bash
# Crear nueva rama desde main
git checkout main
git pull origin main
git checkout -b feature/nombre-descriptivo

# O para bug fixes
git checkout -b fix/descripcion-del-bug

# O para documentación
git checkout -b docs/mejora-documentacion

    Convenciones de Naming#

    • Features: feature/auth-improvements
    • Bug fixes: fix/payment-validation-error
    • Documentation: docs/api-examples-update
    • Refactoring: refactor/user-service-cleanup
    • Tests: test/order-integration-tests

    4. Estándares de Código#

    Estructura de Commits#

    Seguimos Conventional Commits:

    <tipo>[scope opcional]: <descripción>

[cuerpo opcional]

[footer opcional]

    Tipos permitidos: - feat: Nueva funcionalidad - fix: Corrección de bug - docs: Cambios en documentación - style: Formateo (sin cambios de lógica) - refactor: Refactoring de código - test: Agregar o modificar tests - chore: Mantenimiento (dependencias, build, etc.)

    Ejemplos: 

    git commit -m "feat(api): add pagination to cupcakes endpoint"
git commit -m "fix(auth): resolve token expiration handling"
git commit -m "docs(readme): update installation instructions"

    Code Style#

    JavaScript/TypeScript: 

    // ✅ Bueno
const getCupcakes = async (filters = {}) => {
    const { page = 1, limit = 20 } = filters;

    try {
        const response = await api.get('/cupcakes', {
            params: { page, limit }
        });

        return response.data;
    } catch (error) {
        logger.error('Failed to fetch cupcakes', error);
        throw new APIError('Unable to fetch cupcakes', error);
    }
};

// ❌ Malo
function getCupcakes(filters){
  const page = filters.page || 1
  const limit = filters.limit || 20
  return api.get('/cupcakes?page='+page+'&limit='+limit).then(r=>r.data).catch(e=>console.log(e))
}

    API Design: 

    // ✅ Bueno - RESTful y consistente
GET    /api/v1/cupcakes              // Listar cupcakes
POST   /api/v1/cupcakes              // Crear cupcake
GET    /api/v1/cupcakes/{id}         // Obtener cupcake específico
PUT    /api/v1/cupcakes/{id}         // Actualizar cupcake
DELETE /api/v1/cupcakes/{id}         // Eliminar cupcake

// ❌ Malo - No RESTful
GET    /api/getCupcakes
POST   /api/createNewCupcake
GET    /api/cupcake_details?id=123

    5. Testing Requirements#

    Tests Obligatorios#

    Toda contribución debe incluir tests apropiados:

    // Tests unitarios
describe('CupcakeService', () => {
    test('should create cupcake with valid data', async () => {
        const cupcakeData = {
            name: 'Test Cupcake',
            price: 3.50,
            category: 'classic'
        };

        const result = await cupcakeService.create(cupcakeData);

        expect(result.id).toBeDefined();
        expect(result.name).toBe(cupcakeData.name);
        expect(result.price).toBe(cupcakeData.price);
    });

    test('should throw error with invalid data', async () => {
        const invalidData = { name: '' };

        await expect(cupcakeService.create(invalidData))
            .rejects
            .toThrow('Name is required');
    });
});

// Tests de integración
describe('Cupcakes API Integration', () => {
    test('GET /cupcakes should return paginated results', async () => {
        const response = await request(app)
            .get('/api/v1/cupcakes?page=1&limit=5')
            .expect(200);

        expect(response.body.data.cupcakes).toHaveLength(5);
        expect(response.body.pagination).toBeDefined();
    });
});

    Coverage Requirements#

    • Nuevas funcionalidades: 90%+ coverage
    • Bug fixes: 100% coverage de la línea afectada
    • Refactoring: Mantener o mejorar coverage existente

    README Updates#

    Si tu contribución afecta el uso público:

    ## Nuevas Funcionalidades

### Filtrado Avanzado de Cupcakes
Ahora puedes filtrar cupcakes por múltiples criterios:

```javascript
const cupcakes = await cupcakeAPI.getCupcakes({
    category: 'premium',
    allergens: { exclude: ['nuts', 'dairy'] },
    price: { min: 2.00, max: 5.00 },
    rating: { min: 4.5 }
});

    Parámetros disponibles: - category: Filtrar por categoría - allergens.exclude: Excluir alérgenos específicos - price.min/max: Rango de precios - rating.min: Calificación mínima

    🎯 Guidelines Específicos#

    API Design Guidelines#

    1. RESTful Principles#

    // ✅ Bueno
GET    /orders/{orderId}/items          // Obtener items de un pedido
POST   /orders/{orderId}/items          // Agregar item a pedido
PUT    /orders/{orderId}/items/{itemId} // Actualizar item específico

// ❌ Malo
GET    /getOrderItems?orderId=123
POST   /addItemToOrder
PUT    /updateOrderItem

    2. Consistent Response Format#

    // ✅ Bueno - Formato consistente
{
    "status": "success",
    "data": {
        "cupcakes": [...],
        "pagination": {
            "current_page": 1,
            "total_pages": 10,
            "total_items": 200
        }
    }
}

// Para errores
{
    "status": "error",
    "error": {
        "code": "VALIDATION_ERROR",
        "message": "Invalid input data",
        "details": "Price must be greater than 0"
    }
}

    3. Versioning#

    // ✅ Bueno - Versionado en URL
https://api.cupcakeapi.com/v1/cupcakes
https://api.cupcakeapi.com/v2/cupcakes

// Headers para versiones específicas
Accept: application/vnd.cupcakeapi.v1+json

    Security Guidelines#

    1. Input Validation#

    // ✅ Bueno
const createCupcake = async (req, res) => {
    const schema = Joi.object({
        name: Joi.string().min(3).max(100).required(),
        price: Joi.number().positive().max(1000).required(),
        category: Joi.string().valid('classic', 'premium', 'special').required()
    });

    const { error, value } = schema.validate(req.body);
    if (error) {
        return res.status(400).json({
            status: 'error',
            error: {
                code: 'VALIDATION_ERROR',
                message: error.details[0].message
            }
        });
    }

    // Procesar datos validados...
};

    2. Authentication#

    // ✅ Bueno - Middleware de autenticación
const authenticateToken = (req, res, next) => {
    const authHeader = req.headers['authorization'];
    const token = authHeader && authHeader.split(' ')[1];

    if (!token) {
        return res.status(401).json({
            status: 'error',
            error: {
                code: 'AUTH_TOKEN_REQUIRED',
                message: 'Access token is required'
            }
        });
    }

    jwt.verify(token, process.env.JWT_SECRET, (err, user) => {
        if (err) {
            return res.status(403).json({
                status: 'error',
                error: {
                    code: 'AUTH_TOKEN_INVALID',
                    message: 'Invalid or expired token'
                }
            });
        }

        req.user = user;
        next();
    });
};
