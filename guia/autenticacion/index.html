    Autenticación#

    CupcakeAPI utiliza un sistema de autenticación robusto basado en JWT (JSON Web Tokens) y API Keys para garantizar la seguridad de tus datos y transacciones.

    Tipos de Autenticación#

    1. API Key (Para Desarrolladores)#

    Las API Keys se utilizan para identificar a los desarrolladores y sus aplicaciones. Son ideales para: - Aplicaciones server-to-server - Scripts automatizados - Integraciones backend

    2. JWT Tokens (Para Usuarios Finales)#

    Los JWT tokens se utilizan para autenticar usuarios finales de tu aplicación. Son perfectos para: - Aplicaciones web - Aplicaciones móviles - SPAs (Single Page Applications)

    Configuración de API Key#

    Obtener tu API Key#

    1. Regístrate en portal.cupcakeapi.com
    2. Crea un nuevo proyecto
    3. Genera una API Key desde el dashboard

    Usar la API Key#

    Include tu API Key en el header Authorization de todas las peticiones:

    Authorization: Bearer YOUR_API_KEY_HERE

    Ejemplo de Uso#

    const API_KEY = 'ck_live_1234567890abcdef';

fetch('https://api.cupcakeapi.com/v1/cupcakes', {
    headers: {
        'Authorization': `Bearer ${API_KEY}`,
        'Content-Type': 'application/json'
    }
});

    import requests

API_KEY = 'ck_live_1234567890abcdef'

headers = {
    'Authorization': f'Bearer {API_KEY}',
    'Content-Type': 'application/json'
}

response = requests.get('https://api.cupcakeapi.com/v1/cupcakes', headers=headers)

    curl -H "Authorization: Bearer ck_live_1234567890abcdef" \
     https://api.cupcakeapi.com/v1/cupcakes

    Autenticación de Usuarios JWT#

    Registro de Usuario#

    Para registrar un nuevo usuario en tu aplicación:

    POST /auth/register
Content-Type: application/json

{
    "email": "usuario@ejemplo.com",
    "password": "password123",
    "name": "María García",
    "phone": "+1234567890"
}

    Respuesta: 

    {
    "status": "success",
    "data": {
        "user": {
            "id": 12345,
            "email": "usuario@ejemplo.com",
            "name": "María García",
            "phone": "+1234567890",
            "created_at": "2024-01-15T10:30:00Z"
        },
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "expires_in": 3600
    }
}

    Inicio de Sesión#

    Para autenticar un usuario existente:

    POST /auth/login
Content-Type: application/json

{
    "email": "usuario@ejemplo.com",
    "password": "password123"
}

    Respuesta: 

    {
    "status": "success",
    "data": {
        "user": {
            "id": 12345,
            "email": "usuario@ejemplo.com",
            "name": "María García",
            "role": "customer"
        },
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "refresh_token": "rt_1234567890abcdef",
        "expires_in": 3600
    }
}

    Usar el JWT Token#

    Una vez obtenido el token, inclúyelo en el header Authorization:

    Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

    Renovar Token#

    Los tokens JWT tienen una duración limitada. Para renovarlos:

    POST /auth/refresh
Content-Type: application/json

{
    "refresh_token": "rt_1234567890abcdef"
}

    Niveles de Acceso#

    Roles de Usuario#

    CupcakeAPI maneja diferentes niveles de acceso:

    Rol Descripción Permisos
    guest Usuario no autenticado Solo lectura de catálogo público
    customer Cliente registrado Realizar pedidos, ver historial
    employee Empleado de la tienda Gestionar pedidos, inventario básico
    manager Gerente Gestión completa, reportes
    admin Administrador Acceso total al sistema

    Verificar Permisos#

    Puedes verificar los permisos del usuario actual:

    GET /auth/me
Authorization: Bearer YOUR_JWT_TOKEN

    Respuesta: 

    {
    "status": "success",
    "data": {
        "user": {
            "id": 12345,
            "email": "usuario@ejemplo.com",
            "name": "María García",
            "role": "customer",
            "permissions": [
                "order.create",
                "order.view_own",
                "profile.edit"
            ]
        }
    }
}

    Logout y Seguridad#

    Cerrar Sesión#

    Para invalidar un token:

    POST /auth/logout
Authorization: Bearer YOUR_JWT_TOKEN

    Cambiar Contraseña#

    PUT /auth/password
Authorization: Bearer YOUR_JWT_TOKEN
Content-Type: application/json

{
    "current_password": "password123",
    "new_password": "newpassword456"
}

    Recuperar Contraseña#

    POST /auth/forgot-password
Content-Type: application/json

{
    "email": "usuario@ejemplo.com"
}

    Manejo de Errores de Autenticación#

    Códigos de Error Comunes#

    Código Descripción Solución
    401 Token inválido o expirado Renovar o reautenticar
    403 Permisos insuficientes Verificar rol del usuario
    422 Credenciales incorrectas Validar email/password

    Ejemplo de Error#

    {
    "status": "error",
    "error": {
        "code": "TOKEN_EXPIRED",
        "message": "El token JWT ha expirado",
        "details": "Utiliza el refresh_token para obtener uno nuevo"
    }
}

    Mejores Prácticas de Seguridad#

    Para Desarrolladores#

    1. Nunca expongas tu API Key en código cliente
    2. Usa variables de entorno para credenciales
    3. Implementa rate limiting en tu aplicación
    4. Rota las API Keys regularmente

    Para Aplicaciones Cliente#

    1. Almacena tokens de forma segura (Keychain, Secure Storage)
    2. Implementa auto-refresh de tokens
    3. Maneja errores 401 graciosamente
    4. Usa HTTPS siempre

    Ejemplo de Implementación Segura#

    class CupcakeAPIAuth {
    constructor(apiKey) {
        this.apiKey = apiKey;
        this.token = localStorage.getItem('cupcake_token');
        this.refreshToken = localStorage.getItem('cupcake_refresh_token');
    }

    async makeRequest(url, options = {}) {
        // Agregar headers de autenticación
        const headers = {
            'Authorization': `Bearer ${this.apiKey}`,
            'Content-Type': 'application/json',
            ...options.headers
        };

        // Si hay un token de usuario, usarlo en su lugar
        if (this.token) {
            headers.Authorization = `Bearer ${this.token}`;
        }

        try {
            const response = await fetch(url, { ...options, headers });

            // Si el token expiró, intentar renovarlo
            if (response.status === 401 && this.refreshToken) {
                await this.refreshAuthToken();
                headers.Authorization = `Bearer ${this.token}`;
                return fetch(url, { ...options, headers });
            }

            return response;
        } catch (error) {
            console.error('Error en petición:', error);
            throw error;
        }
    }

    async refreshAuthToken() {
        const response = await fetch('/auth/refresh', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ refresh_token: this.refreshToken })
        });

        if (response.ok) {
            const data = await response.json();
            this.token = data.data.token;
            localStorage.setItem('cupcake_token', this.token);
        } else {
            // Si no se puede renovar, limpiar tokens
            this.clearTokens();
        }
    }

    clearTokens() {
        this.token = null;
        this.refreshToken = null;
        localStorage.removeItem('cupcake_token');
        localStorage.removeItem('cupcake_refresh_token');
    }
}

    Con esta configuración de autenticación, tu aplicación estará lista para manejar usuarios de forma segura. Continúa con la documentación de endpoints para explorar las funcionalidades disponibles.
