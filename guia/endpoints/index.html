    Endpoints de la API#

    Esta sección proporciona una referencia completa de todos los endpoints disponibles en CupcakeAPI, organizados por funcionalidad.

    Base URL#

    https://api.cupcakeapi.com/v1

    Autenticación#

    Todos los endpoints requieren autenticación excepto los marcados como públicos. Ver Guía de Autenticación.

    🧁 Cupcakes#

    Listar Cupcakes#

    Obtiene una lista paginada de todos los cupcakes disponibles.

    GET /cupcakes

    Parámetros de Query: - page (int): Número de página (default: 1) - limit (int): Elementos por página (default: 20, max: 100) - category (string): Filtrar por categoría (clasico, premium, especial, temporada) - available (boolean): Solo cupcakes disponibles (default: true) - search (string): Buscar por nombre o descripción - sort (string): Ordenar por (name, price, popularity, created_at) - order (string): Dirección (asc, desc)

    Ejemplo de Petición: 

    curl -H "Authorization: Bearer YOUR_API_KEY" \
     "https://api.cupcakeapi.com/v1/cupcakes?category=premium&limit=10"

    Obtener Cupcake Específico#

    Obtiene los detalles completos de un cupcake específico.

    GET /cupcakes/{id}

    Parámetros de Ruta: - id (int): ID del cupcake

    Respuesta: 

    {
    "status": "success",
    "data": {
        "cupcake": {
            "id": 1,
            "name": "Red Velvet Supreme",
            // ... todos los campos del cupcake
            "related_cupcakes": [2, 5, 8],
            "recipe": {
                "instructions": [
                    "Precalentar horno a 180°C",
                    "Mezclar ingredientes secos...",
                    "Batir claras a punto de nieve..."
                ],
                "baking_time": 18,
                "baking_temperature": 180
            }
        }
    }
}

    Crear Cupcake#

    Crea un nuevo cupcake en el catálogo. Requiere permisos de manager o superior.

    POST /cupcakes
Authorization: Bearer JWT_TOKEN

    Listar Pedidos del Usuario#

    Obtiene el historial de pedidos del usuario autenticado.

    GET /orders
Authorization: Bearer JWT_TOKEN

    Parámetros de Query: - status (string): Filtrar por estado (pending, confirmed, preparing, ready, delivered, cancelled) - page (int): Número de página - limit (int): Elementos por página

    Obtener Pedido Específico#

    GET /orders/{order_id}
Authorization: Bearer JWT_TOKEN

    Cancelar Pedido#

    DELETE /orders/{order_id}
Authorization: Bearer JWT_TOKEN

    Respuesta: 

    {
    "status": "success",
    "message": "Pedido cancelado exitosamente",
    "data": {
        "refund": {
            "amount": 66.96,
            "method": "original_payment_method",
            "estimated_time": "3-5 días hábiles"
        }
    }
}

    📊 Reportes y Analytics#

    Dashboard de Métricas#

    GET /analytics/dashboard
Authorization: Bearer JWT_TOKEN

    🏪 Gestión de Inventario#

    Listar Inventario#

    GET /inventory
Authorization: Bearer JWT_TOKEN

    Actualizar Stock#

    PATCH /inventory/{cupcake_id}/stock
Authorization: Bearer JWT_TOKEN
Content-Type: application/json

    Cuerpo de la Petición:

    "quantity": 50,
    "operation": "set", // "set", "add", "subtract"
    "reason": "Restock semanal"
}

    📱 Webhooks#

    Configurar Webhook#

    POST /webhooks
Authorization: Bearer API_KEY
Content-Type: application/json

    Cuerpo de la Petición: 

    {
    "url": "https://tu-aplicacion.com/webhooks/cupcakeapi",
    "events": [
        "order.created",
        "order.status_changed",
        "payment.completed",
        "inventory.low_stock"
    ],
    "secret": "tu_secreto_webhook"
}

    Códigos de Estado HTTP#

    Código Descripción
    200 OK - Petición exitosa
    201 Created - Recurso creado exitosamente
    204 No Content - Petición exitosa sin contenido
    400 Bad Request - Error en los parámetros de la petición
    401 Unauthorized - Token inválido o faltante
    403 Forbidden - Permisos insuficientes
    404 Not Found - Recurso no encontrado
    422 Unprocessable Entity - Errores de validación
    429 Too Many Requests - Rate limit excedido
    500 Internal Server Error - Error interno del servidor

    Rate Limiting#

    • Desarrollo: 100 requests/minuto
    • Producción: 1000 requests/minuto
    • Enterprise: Sin límites

    Los headers de respuesta incluyen información sobre el rate limiting:

    X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1640995200
