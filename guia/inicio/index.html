    Guía de Inicio#

    Bienvenido a CupcakeAPI. Esta guía te llevará paso a paso desde la configuración inicial hasta realizar tu primera llamada exitosa a la API.

    Requisitos Previos#

    Antes de comenzar, asegúrate de tener:

    • Conocimientos básicos de APIs REST
    • Una herramienta para realizar peticiones HTTP (cURL, Postman, Insomnia)
    • Un lenguaje de programación de tu preferencia (JavaScript, Python, PHP, etc.)

    Paso 1: Registro y Obtención de Credenciales#

    1.1 Crear una Cuenta de Desarrollador#

    1. Visita portal.cupcakeapi.com
    2. Haz clic en "Registrarse como Desarrollador"
    3. Completa el formulario con tu información: 
      Nombre: Tu nombre completo
Email: tu@email.com
Empresa: Nombre de tu empresa (opcional)
Tipo de aplicación: Web, Móvil, Desktop, etc.

    1.2 Verificación de Email#

    Después del registro, recibirás un email de verificación. Haz clic en el enlace para activar tu cuenta.

    1.3 Generar API Key#

    1. Inicia sesión en el portal de desarrolladores
    2. Ve a "API Keys" en el menú lateral
    3. Haz clic en "Generar Nueva Key"
    4. Copia y guarda tu API Key de forma segura

    Importante

    Tu API Key es secreta. No la compartas públicamente ni la incluyas en repositorios de código abierto.

    Paso 2: Configuración del Entorno#

    Base URL#

    Todas las peticiones deben dirigirse a: 

    https://api.cupcakeapi.com/v1/

    Headers Requeridos#

    Authorization: Bearer YOUR_API_KEY
Content-Type: application/json
Accept: application/json

    Ejemplo de Configuración#

    const API_BASE_URL = 'https://api.cupcakeapi.com/v1';
const API_KEY = 'tu_api_key_aqui';

const headers = {
    'Authorization': `Bearer ${API_KEY}`,
    'Content-Type': 'application/json',
    'Accept': 'application/json'
};

    Paso 3: Tu Primera Llamada a la API#

    Vamos a obtener la lista de cupcakes disponibles:

    async function getCupcakes() {
    try {
        const response = await fetch(`${API_BASE_URL}/cupcakes`, {
            method: 'GET',
            headers: headers
        });

        if (!response.ok) {
            throw new Error(`HTTP error! status: ${response.status}`);
        }

        const cupcakes = await response.json();
        console.log('Cupcakes disponibles:', cupcakes);
        return cupcakes;
    } catch (error) {
        console.error('Error al obtener cupcakes:', error);
    }
}

getCupcakes();

    def get_cupcakes():
    try:
        response = requests.get(f"{API_BASE_URL}/cupcakes", headers=headers)
        response.raise_for_status()

        cupcakes = response.json()
        print("Cupcakes disponibles:", cupcakes)
        return cupcakes
    except requests.exceptions.RequestException as e:
        print(f"Error al obtener cupcakes: {e}")

    Paso 4: Manejo de Errores#

    Si algo sale mal, la API devuelve códigos de estado HTTP estándar:

    {
    "status": "error",
    "error": {
        "code": "INVALID_API_KEY",
        "message": "La API key proporcionada no es válida",
        "details": "Verifica que hayas copiado correctamente tu API key"
    }
}

    Consejos para Empezar#

    Rate Limiting#

    • Desarrollo: 100 requests/minuto
    • Producción: 1000 requests/minuto
    • Enterprise: Sin límites

    Mejores Prácticas#

    1. Cachea las respuestas cuando sea posible
    2. Implementa retry logic para errores temporales
    3. Usa paginación para listas grandes
    4. Maneja errores graciosamente
    5. Mantén tus API keys seguras

    Herramientas Recomendadas#

    • Postman Collection: Importar colección
    • OpenAPI Spec: Descargar spec
    • SDKs Oficiales: JavaScript, Python, PHP, Ruby

    ¿Listo para continuar? Dirígete a la documentación de endpoints para explorar todas las funcionalidades disponibles.
